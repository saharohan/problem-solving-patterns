# Problem Solving Patterns

A collection of problem-solving patterns and notes while revisiting DSA fundamentals and interview-oriented problems.

The goal here is not to optimize for question count, but to understand patterns, improve reasoning, and write clean implementations.

This repository is maintained as a structured reference for commonly used approaches, data structures and problem-solving intuition.

## What you'll find here

* 📌 Pattern-based problem organization
* ⚙️ Key idea, intuition and approach
* ⏱️ Time and space complexity analysis
* 💻 Java implementations
* 🏷️ Patterns and implementation notes where relevant

## Structure

* 📂 arrays-hashing
* 📂 sliding-window
* 📂 two-pointers
* 📂 stack
* 📂 binary-search
* 📂 heap-priority-queue
* 📂 trees
* 📂 graphs
* 📂 intervals


Each problem generally contains:

* short problem summary
* key idea / intuition
* why the approach works
* complexity analysis
* implementation
* patterns or implementation notes where relevant

## Why this repo

Revisiting problem-solving patterns helps improve reasoning, strengthen data structure intuition and prepare for technical interviews.

It also acts as a quick reference for commonly used approaches and reusable patterns.

## Notes

* Problems are grouped by pattern instead of platform order
* Preference is given to readability and clarity over overly clever code
* Focus is on understanding patterns and reasoning rather than memorizing solutions

**Language:** Java
